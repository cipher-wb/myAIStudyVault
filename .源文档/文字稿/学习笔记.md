#             **1.**     **Git基础概念**

 



**提醒事项** 

请确保你的电脑已经安装了Git！

# **Git和Github的区别**

##  

## **Git**

![img](https://wdcdn.qpic.cn/MTMxMDI3MDE1NDg0OTIxMzA_232995_FMJLo206ak3nlBM0_1773220975?w=2866&h=1210)

 

​                ● **定义：**一个分布式版本控制系统

​                ● **功能：**追踪文件变化历史，管理代码版本

​                ● **工作方式：**在本地创建代码仓库，记录所有更改

​                ● **主要特点：**

​                ○ 完全离线工作，不需要网络连接

​                ○ 快速的分支创建和合并

​                ○ 强大的历史记录和版本回溯能力

​                ● **使用场景：**单人开发或团队协作的代码版本管理

 

## **Github**

![img](https://wdcdn.qpic.cn/MTMxMDI3MDE1NDg0OTIxMzA_645443_gqCfLKelFjQiW30W_1773220975?w=3694&h=1812)

​                ● **定义：**基于Git的云端托管服务

​                ● **功能：**远程存储Git仓库并提供协作工具

​                ● **工作方式：**提供云端存储空间和Web界面来管理Git仓库

​                ● **主要特点：**

​                ○ 远程代码仓库托管

​                ○ Pull Requests（拉取请求）实现代码审查

​                ○ Issues（问题）追踪系统

​                ○ Actions（自动化工作流）

​                ○ 社区和协作功能

​                ● **使用场景：**团队协作、开源项目、代码分享

 

# **Git常见命令**

 

Git 是一个版本控制系统，帮助你跟踪代码的变化。以下是日常使用的核心命令：

 



**Git最重要的几个命令：**

​                ● **git init**：初始化一个新的Git仓库

​                ● **git clone**：克隆现有的仓库

​                ● **git add** ： 将文件添加到暂存区

​                ● **git commit**：提交暂存区的更改

​                ● **git push** ：推送本地更改到远程仓库

​                ● **git status** ：查看工作区状态

​                ● **git pull** ：拉取远程更改并合并

​                ● **git branch** ：查看、创建或删除分支

​                ● **git checkout** ：切换

​                ● **git switch**：切换分支

​                ● **git merge** ：合并分支





 

你目前可能不知道这些命令是干什么，没关系。接下来我会带你去认识这些操作：

​            1.     你可以完全使用UI，不去了解这些命令的具体含义

​            2.     你也可以像一个极客一样使用这些命令。

## **4. 分支管理**



**为什么要有分支？** 

想象这么一个场景，如果你的项目上线了。你现在正在开发新的功能，开发到一半的时候，发现线上有一个非常紧急的bug需要处理。那这个时候你就要在当前开发到一半的分支的基础上去处理这个bug。

 

那问题来了，如果说你处理完这个bug，然后把这个项目推送上线的话，那你同时也把你这开发到一半的功能推送上线给用户了。

 

那应该怎么解决这个问题呢？这个就需要分支来解决！

点击【分支】-【创建分支】，你就可以创建你的分支名字

#             **3.**     **基于git命令的教学实战**

 

## **创建和获取仓库**

 

### **git init****：开始一个新项目**

 



```
git init 
```





这会在当前文件夹创建一个新的 Git 仓库。

 

### **git clone****：获取已有项目**

 



```
git clone https://github.com/用户名/项目名.git 这会下载一个完整的项目副本到你的电脑。
```

 



 

## **日常工作流程**

 

### **git status****：查看当前状态**

 



```
 git status 
```





显示哪些文件被修改、暂存或未跟踪。

###  

### **git add****：添加暂存区**

 



```
git add 文件名    # 添加单个文件 
git add .        # 添加所有更改 
```

 



将文件放入"暂存区"，准备提交。

###  

### **git commit****：提交更改**

 



```
git commit -m "描述你做了什么更改" 
```

 



将暂存区的内容保存为一个新的提交记录。

###  

### **git log****：查看提交日志**



```
 git log 
```





查看你所有的提交的日志，会显示你git commit的信息。

##  

## **远程协作**

 

### **git push****：上传更改**

 



```
git push origin main 
```

 



将本地提交推送到远程仓库（例如 GitHub）。

 

### **git pull****：获取更新**



```
git pull 
```

 



从远程仓库下载更新并合并到本地。

 

## **分支管理**

 

### **git branch****：分支操作**

 

### `git branch           # 查看分支列表 git branch 新分支名   # 创建新分支 git branch -d 分支名  # 删除分支 ` 

###  

### **git checkout****：检出与切换**

 



```
git checkout 提交ID     # 查看历史版本 
git checkout 分支名      # 切换到某分支（官方不再推荐，但依然大规模使用） 
git checkout -b 新分支名 # 创建并切换到新分支（官方不再推荐，但依然大规模使用） 
```

 



###  

### **git switch****：现代分支切换（新版Git）**

 

### `git switch 分支名       # 切换分支 git switch -c 新分支名  # 创建并切换到新分支 ` 

###  

### **git merge****：合并分支**



```
git switch main       # 先切到目标分支 
git merge 特性分支名    # 将特性分支合并到当前分支
```





##   

##  

## **退回某个版本**

 

### **git reset****：退回到某个提交**



```
git reset 提交ID 
```

 



##  

## **实用小贴士**

 

​                ● 使用 git status 随时检查工作状态

​                ● 经常提交，提交信息要有意义

​                ● 使用分支进行新功能开发

​                ● 合并前先拉取最新代码

 

这些基本命令可以满足大部分日常开发需求，熟练掌握后会让你的开发工作更高效。
